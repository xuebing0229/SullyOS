# 忠实用户一次性招募

> 一次性历史截面检测。资格完全在用户本机计算；未通过者的 QQ 不上传，通过者只向独立招募 Worker 登记 QQ，供新群成员名单做差集排查。

## 用户入口与迁移说明

- 不在启动或解锁时主动弹窗。
- 入口位于「设置」页面最底部的低调文字项「社区迁移说明」。
- 说明页明确告知：Discord 社区已经扩大并承载全部社区功能；QQ 只保留较小的日常水群。作者不再承担 3000 人群的扩容费用，改用更低人数规格，因此新的 QQ 群限制进入人数。
- 未通过页面不做负面评价，并引导用户前往完整开放的 Discord 社区：`https://discord.gg/rvqBaTUgR`。

## 时间口径

- 数据截止：`2026-07-20T19:00:00+08:00`。
- 最近一个月：`2026-06-20T19:00:00+08:00` 至截止点。
- 所有时间比较显式使用北京时间，不跟随设备时区。
- `MemoryFragment.date` 只有日期没有时分，因此排除 7 月 20 日整天；`refinedMemories` 的 2026 年 7 月总结同样排除。

## 判定

总分 100，65 分通过：最近一个月活跃 50、非默认角色 15、神经链接记忆 25、记忆宫殿可选加分 10。

最近一个月还有硬门槛：至少 4 个活跃日、分布在至少 2 个北京时间自然周、至少 20 条用户有效消息。活跃日要求当天至少 3 条用户有效消息。系统消息、角色主动消息和卡片不计。

另设固定的「深度用户通道」：首先必须存在 Sully 以外、且在截止日前有使用证据的角色；随后满足以下任意一项即可直接通过：任意单个角色在截止日前累计至少 3 个活跃日；7 月 1 日至截止日前至少 3 条有效神经链接记忆；6 月 1 日至截止日前至少 15 条有效神经链接记忆；截止日前累计至少 200 条有效神经链接记忆；或记忆宫殿有效节点超过 20。这里的活跃日仍要求当天至少 3 条用户有效消息，但不限制在最近一个月。神经链接门槛按实际记忆条目数计算，月度精炼总结不冒充单条记忆。

规则升级到 `2026-07-20-v4` 时，旧版失败用户优先只用已经封存的指标重新分类，不读取截止日后的数据。旧摘要没有“单个角色历史活跃日”和 6 月 / 7 月分段条目数字段时，以已封存的近期活跃日、6 月 20 日后记忆量作宽松兼容证据；符合深度通道的旧结果进入待填写 QQ 状态，其余失败结果保持不变。

详细阈值与过滤规则的单一来源是 `utils/loyalUserEligibility.ts`，界面不展示分数或未通过原因。

## 一次性状态

状态存在 `localStorage`：

```text
sullyos_loyal_recruitment_2026-07-20-v4
```

- `failed` / `registered`：从设置入口再次打开时只展示已经封存的结果，不重新评分。
- `declined`：兼容旧版本遗留状态；新的设置入口关闭时不再写入该状态。
- `passed_pending`：本地已经通过，但 QQ 尚未成功登记；只能重试登记，不重新评分。
- `not_selected`：最终名额状态已确认但未分配；QQ 不进入登记名单，界面只展示通用的未获取名额说明。
- IndexedDB 或其它本地读取失败不写状态，不消耗机会。

开发测试时打开 Dev Debug，点击「迁移重测」会删除当前及兼容旧版 key 并刷新；刷新后需重新进入「设置 → 社区迁移说明」。正式构建默认看不到重测入口。

## Worker

独立服务位于 `worker/loyal-recruitment`，不与邮局或彼方活动共享 Worker、D1 或 secrets：

- `POST /submit`：登记通过者 QQ，返回群号和密码。
- 迁移期间同时接受 `2026-07-20-v1` 至 `v4` 封存凭证，避免前端与 Worker 发布顺序造成短暂登记失败。
- `GET /admin`：管理员网页；输入 `ADMIN_TOKEN` 后查看名单并下载 CSV。
- `GET /admin-list`：Bearer `ADMIN_TOKEN` 导出登记名单。
- `recruit_candidates`：独立 D1 表。
- 默认前端地址：`https://noir2.cc.cd/recruit`。

部署前必须配置：

```bash
wrangler secret put GROUP_PASSWORD
```

群号使用 `GROUP_ID`；入群密码不写进仓库。正式域名尚未绑定时，可在本机把
`sullyos_loyal_recruitment_base` 覆盖成该 Worker 的 `workers.dev` 地址进行验收。
