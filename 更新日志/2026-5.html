01

新增 · GitHub 数据备份 NEW

把存档放进自己的 GitHub 仓库，还不用挂梯子。

数据备份现在可以直接走 GitHub。在设置 App›数据里能看到新的云端数据区域，按指引一步步接入即可。

  • 云端数据这部分接的是 GitHub 仓库，不再需要梯子就能正常用。
  • 每一次备份都对应一次仓库提交，历史记录可追溯。
  • 换设备 / 重装时，可以直接从 GitHub 把数据拉回来。
配置位置

设置›数据›云端数据（GitHub） ——按界面里的指引一步步授权和填写即可。

02

优化 · 音乐 App 弱化梯子依赖 POLISH

不挂梯子也能听一段。

调整了音乐 App 的网络与资源链路，对梯子的要求明显减弱，一定程度上可以裸连使用。

  • 常规听歌、封面加载等场景在国内网络下更稳。
  • 极少数仍需要梯子的资源会自动回退或提示。
  • 如果之前因为网络一直打不开音乐 App，这次值得再试一次。
03

新增 · 麦当劳 MCP NEW

让角色帮你点一份麦当劳。

新增麦当劳 MCP能力，把点餐变成角色可以参与的一段日常。开启后，你可以让角色一起翻菜单、推荐套餐、最终下到一份属于你们俩的订单。

开启位置

设置 App›麦当劳 MCP ——按设置内的提示一步步配置好之后即可使用。

使用方式

  1. 进入任意角色的聊天界面。
  2. 点击聊天输入框旁的 + 加号，滑动到第二页。
  3. 找到「麦当劳点餐」入口，点击进入点餐界面。
温馨提示

第一次使用记得回到 设置 App 完成配置。没配置好的话加号里的入口会无法正常工作。

04

新增 · HTML 模块 NEW

让 AI 输出可渲染的小卡片。

新增HTML 模块，让聊天里能直接出现一段经过渲染的 HTML 小卡片，无论是漂亮的列表、徽章、还是一张迷你海报，都能让对话显得更立体。

开启位置

聊天界面 + 加号第二页，紧挨着「麦当劳点餐」功能的旁边即可看到「HTML」入口。

  • 点开后会进入 HTML 模块的快捷开关与说明。
  • 开启后，AI 输出的 [html]...[/html] 卡片会被渲染成可视化样式。
  • 关闭后，原文会以普通文本形式呈现，不会丢失内容。
05

新增 · 写歌生成音乐 NEW

让那些被写下的歌词，真的开始唱起来。

新增写歌生成音乐能力。以前你和角色在「写歌」里反复修改的歌词，只能停在文字阶段；现在它们可以被直接生成成一段可播放的音乐。

  • 角色（char）写出的歌词同样支持一键生成。
  • 生成结果会回到写歌作品里，可随时回听。
  • 风格和情绪可以在歌词里点拨，让生成更贴你想要的样子。
小提醒

音乐生成依赖外部能力，第一次使用前请先确认对应配置已就绪。

06

新增 · API 流式开关 & 温度调整 NEW

给更高阶的玩家留一道小门。

在 API 配置里追加了流式开关与温度调整两项。它们藏在「高级」折叠区里，默认收起、字色偏灰，并明确标注「不建议主动修改」。

  • 流式开关：控制是否走流式输出。绝大多数模型保持默认即可。
  • 温度：影响回复的随机度。改动会显著改变角色风格，慎调。
面向高阶用户

如果你不知道这两项是干什么的，请保持默认。它们更像"留给折腾党"的开关，新手碰了反而容易把回复弄坏。

07

新增 · SULLY 默认外观皮肤 NEW

把 SULLY 原本的样子，做成了一个可一键复原的预设。

外观›预设 里追加了一份 SULLY 默认外观皮肤。无论你之前怎么折腾过壁纸、配色、字体，都可以一键回到最初的 SULLY 模样。

  • 位置：外观 App → 预设列表，名字就是 SULLY 默认外观。
  • 点击应用即可恢复，原有自定义不会被立刻覆盖到不可逆。
  • 适合"折腾完想回家"或者"不知道选什么先用着"的场景。

新增 · Instant Push NEW

独立的 LLM-driven Web Push 通道：自部署 Worker，LLM 在云端生成回复后分句推送到手机。

引入 @rei-standard/amsg-instant 0.2.0，同步将 amsg-client 升至 2.2.0、amsg-server 升至 2.1.0。 每个 fork 用户自己在 Cloudflare 上部署一个独立 Worker——零数据库、零 cron、跟作者的 sully-n 完全无关。

  • 前端在「设置 → Instant Push」里一键生成 VAPID 密钥对、复制 Worker 代码、填 Worker URL，即可完成接入。
  • 测试推送会真实调用你的 Chat API，把 LLM 生成的一句话通过 Web Push 发到你的设备。
  • 采用明文协议（HTTPS 已加密传输），不引入 masterKey / 加密字段，配置极简。
配置位置

设置›Instant Push›配置，或参考 worker/instant-push/README.md。
