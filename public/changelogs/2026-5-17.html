Instant Push 上线 Send the message, lock the screen, walk away.

这次的主菜是新增的发消息模式 ——「Instant Push」。开了之后，你给角色发完一条文本就可以锁屏走人，AI 的回复会自己以系统推送通知的形式回来。不用一直把 App 留在前台，也不怕被系统省电策略杀后台。

主题新增发送模式 类型小版本更新 阅读约 1 分钟

本次更新概览

  1. 新增 Instant Push 模式：发完文本就能锁屏走人，AI 回复自己回来。
  2. AI 跑在你自己的 Cloudflare Worker 上，App 被杀后台也不打断。
  3. 回复通过系统推送通知送达，点通知直接落到对应角色的聊天里。
  4. 目前只有主聊天框的普通文本消息会触发，其它类型暂不支持。
  5. 需要先配 VAPID 公私钥 + 部署一次 Worker（设置里有逐步指引）。
01

新增 · Instant Push 模式 NEW

发完就走，回复自己回来。

之前是这样

你给角色发了一条消息，必须把 App 留在前台、屏幕亮着，盯着等 AI 一个字一个字流式吐回来。中途切个微信、锁个屏、被系统省电策略杀进程，这一轮回复就断了，回来还得手动重发或重试。

开了 Instant Push 之后

你发完文本，App 把这一轮请求打包丢给云端的 Worker就完事了 —— 你可以直接锁屏、切别的应用、甚至让系统把这个 App 杀掉，都不影响。Worker 在云端默默调你配的 LLM API，跑完之后通过系统级推送通知把回复推回你的设备，消息也会自动写进对应角色的聊天历史。

视觉上能看到什么

  • 你发出去的那条文本气泡左侧会出现三个跳动的小圆点，表示「请求已交给 Worker，回复在路上」。
  • Worker 那边一收到（返回 200）小圆点就消失 —— 这时候你可以放心锁屏了。
  • AI 回复到达时，系统通知栏会弹一条，点进去直接跳到对应聊天。
关于"我的 API Key 安不安全"

Worker 跑在你自己的 Cloudflare 账户下，LLM API 配置、Key 这些都在你自己 Worker 的环境变量里，App 这边只把请求载荷发过去，不经过任何第三方中转。

02

第一版只接了普通聊天文本 SCOPE

先把最高频、最受"杀后台"折磨的场景做掉。

哪些消息会走 Instant Push，哪些不会，先一次说清楚：

  • 主聊天框 · 普通文本消息（目前唯一会走 Instant Push 的路径）
  • 图片消息（走本地多模态路径，没接 Worker）
  • 小红书卡片（XHS 笔记卡片走本地拉取）
  • 麦当劳协同点餐（依赖本地小程序状态同步）
  • 「约会（见面模式）」（自己有一套独立上下文构建，暂未接入）
  • 主动消息 / 心象等彩蛋功能（不在这次范围内）
为什么先只做这一种

Instant Push 第一版只做主聊天框的纯文本，是因为这条路径最高频、最简单、也最受"杀后台"影响 —— 锁屏 / 切 App 的痛点 90% 集中在这里。

其它路径要么依赖本地状态（图片走本地多模态、麦当劳要同步小程序），要么需要更复杂的中间过程（卡片、见面模式有自己的 prompt 组装），接进来需要单独的工程量。先发主路径让大家用上，后面按反馈优先级再补。

不影响原有体验

没配 Instant Push、或者关掉它的时候，所有行为跟之前完全一样 —— 前台手动等回复。即使开了 Instant Push，发图片、卡片、见面模式这些也都自动走老路径，不会发到 Worker，也不会出问题。

03

怎么开启

第一次配大概要十分钟，之后就一劳永逸。
  1. 进入 设置›Instant Push 面板。
  2. 如果还没配过 VAPID 公私钥，先到面板顶部的「推送凭据 (VAPID)」按钮生成一份（与 Proactive Push 共用同一份，配过就不用重复配）。
  3. 按面板里「② 部署 Worker」的指引走 —— 它会给你一个 Git URL，直接在 Cloudflare 后台「Clone a public repository via Git URL」粘进去新建 Worker；env 变量从面板里复制粘贴即可。
  4. 把部署好的 Worker URL 填回 App，勾上「启用 Instant Push」，保存。
  5. 回到聊天界面，下次发文本就自动走 Instant Push —— 看到气泡左侧三个跳动小圆点就对了。
关于 VAPID 这次的调整

这次顺手把 VAPID 公私钥提取成了"全局推送凭据" —— Proactive Push（角色主动给你发消息）和 Instant Push 共用一份，不再各自管各自的。如果你之前两边都分别配过、key 不一致，现在以「推送凭据 (VAPID)」面板里的那份为准，重新订阅一次就行（点测试推送验证一下就知道通没通）。
